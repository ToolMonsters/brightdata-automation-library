CRM Lead Enrichment on New Contact

Every time a new contact lands in your CRM with a LinkedIn URL, Bright Data scrapes their profile, AI extracts title / company / tech stack, the CRM record gets updated, and your sales team gets pinged on Slack. Works with HubSpot, Pipedrive, Salesforce, Attio, and more.

Zapier template coming soon. In the meantime, you can build this automation on Make or n8n using the step-by-step guides in the other tabs.
What this scenario does: When a new contact is created in your CRM, Bright Data scrapes the LinkedIn profile, OpenAI extracts structured data (title, company, seniority, industry, tech stack, location), and the CRM record gets updated automatically.

5 modules: CRM Watch Contacts → Bright Data (Make an API Call) → OpenAI Chat Completion → Parse JSON → CRM Update Contact

Get Make template
Works with any CRM: This scenario uses HubSpot as an example, but you can swap the trigger and update modules with Pipedrive, Salesforce, Attio, Airtable, or any other CRM supported by Make. The Bright Data + OpenAI logic stays the same.
1
Prepare your CRM properties
Your contact record needs a LinkedIn URL field as input, plus fields to receive the enriched data:
  • Job Title, Company, Seniority Level
  • Industry, City, Country
  • Tech Stack, One-line Summary
2
Set up the trigger (first module) - any CRM works
By default the template uses HubSpot CRM - Watch Contacts. Click the first module, connect your HubSpot account, and make sure hs_linkedin_url is selected in Output Properties.

Using a different CRM? Delete this module and replace it with the equivalent trigger:
  • Pipedrive → Watch Persons
  • Salesforce → Watch Records (Contact object)
  • Attio → Watch Records (People)
  • Airtable → Watch Records
What matters: the trigger must output a LinkedIn URL field. That's the only value Bright Data needs. Everything else (name, email, company) is optional but useful for the Slack notification at the end.
3
Connect the Bright Data module
The second module is Bright Data - Make an API Call. Click it and connect your Bright Data account (API key from your Bright Data settings).

If you swapped the CRM trigger in step 2, you need to re-map the LinkedIn URL in the Body field. Click on the Body field and make sure it points to the LinkedIn URL field from your trigger:
{ "input": [ { "url": "{{LinkedIn URL field from step 1}}" } ] }
Everything else (URL, Method, Query String, Headers) stays the same. Don't touch the dataset_id - it's the LinkedIn People Profiles dataset.
4
Connect OpenAI
The third module is OpenAI - Create a Chat Completion. Click it and connect your OpenAI account (API key from OpenAI platform). The prompt and schema are already pre-configured - you don't need to change anything unless you want to customize the fields extracted.

View the prompt used (for reference)
Configuration:
  • Model → gpt-4o-mini (or gpt-5-mini)
  • Max Output Tokens → 1024
  • Temperature → 1
Add two messages. System message:
You are a LinkedIn profile data extractor. Extract structured data from the LinkedIn profile JSON provided and return ONLY a valid JSON object matching this exact schema. No prose, no commentary, no markdown fences. Schema: { "current_title": "string or null", "current_company": "string or null", "seniority_level": "IC | Manager | Director | VP | C-level | Founder | Unknown", "years_experience_estimate": "number or null", "industry": "string or null", "location_city": "string or null", "location_country": "string or null", "tech_stack": ["array of technologies, tools, programming languages, or domain skills"], "top_certifications": ["array of max 3 most relevant certifications"], "one_line_summary": "string, 1 sentence professional summary" } Rules: - Infer seniority from title, experience, and followers count - Infer tech_stack from posts, experience, certifications, and about sections - Use country codes from country_code field or infer from city - If a field cannot be determined, return null or empty array - Return valid JSON only, no code blocks
User message:
LinkedIn profile data: {{Bright Data response body}}
5
Parse JSON (already configured)
The fourth module is JSON - Parse JSON. It takes the OpenAI response and converts it into structured fields you can map. This is already configured - no action needed unless you changed the OpenAI schema.
6
Update the CRM contact (last module) - any CRM works
The last module is HubSpot CRM - Update a Contact. This is where the enriched data gets written back to your CRM. This is the step you need to customize most.

If you kept HubSpot: Click the module, connect your HubSpot account, and verify that each mapping is correct:
  • Contact ID → from the trigger (step 1)
  • Job Title → current_title from Parse JSON
  • Company → current_company from Parse JSON
  • City / Country → location_city / location_country
  • Industry → industry
  • Custom fields (Seniority, Tech Stack, Summary) → map if you created them
If you swapped the CRM: Delete this module and replace it with the equivalent:
  • Pipedrive → Update a Person
  • Salesforce → Update a Record
  • Attio → Update a Record
  • Airtable → Update a Record
Then map each field from Parse JSON to the matching field in your CRM. Make sure the Contact ID comes from your trigger - not a hardcoded value - so the right record gets updated.
7
(Optional) Add a Slack notification
Add a Slack - Create a Message module. Select your channel (e.g. #sales-leads) and compose a message with the enriched data (name, title, company, seniority, tech stack, LinkedIn URL).
8
Test and activate
Click Run once and add a test contact with a LinkedIn URL. Check that all modules complete green and the contact gets enriched. Toggle the scenario ON.

Troubleshooting

ProblemSolution
401 Unauthorized on Bright DataCheck your Bright Data connection in the module settings.
OpenAI returns non-JSONSet Response Format to json_object in the OpenAI module, or keep the strict system prompt as shown.
Parse JSON failsMake sure you're parsing {{9.choices[].message.content}} (not the full OpenAI response). Check that the custom data structure matches the schema exactly.
CRM trigger not firingIncrease polling frequency in the Watch Contacts module. For HubSpot, verify the API scope includes contacts read/write.
LinkedIn URL emptyAdd a Filter between the trigger and Bright Data: only continue if hs_linkedin_url is not empty.
Want to use a different CRMReplace HubSpot Watch Contacts and Update Contact modules with the equivalent Pipedrive, Salesforce, or Attio modules. The Bright Data + OpenAI + Parse JSON steps don't change.
What this workflow does: A new lead in your CRM triggers a LinkedIn scrape via Bright Data. GPT-5-mini extracts structured data (title, company, seniority, industry, tech stack, summary). The CRM record gets updated and a Slack notification is sent.

5 nodes: CRM Trigger → Bright Data (Scrape LinkedIn Profile) → OpenAI (Extract Structured Data) → CRM Update → Slack Notify

Get n8n template
Works with any CRM or database: the template uses Airtable by default, but you can swap it for HubSpot, Salesforce, Pipedrive, Attio, Close, Notion, Google Sheets, Monday.com, ClickUp, or any database (Postgres, MySQL, MongoDB, Supabase). Only the trigger and update nodes need to change - the rest stays the same.
1
Set up the trigger (first node) - any CRM works
By default the template uses an Airtable Trigger that polls every minute using the Created Time field. Click the node, connect your Airtable account, and fill in your Base ID and Table ID.

Using a different CRM? Delete the Airtable Trigger and replace it with the equivalent:
  • HubSpot → HubSpot Trigger (Contact Created)
  • Pipedrive → Pipedrive Trigger (New Person)
  • Salesforce → Salesforce Trigger
  • Notion → Notion Trigger
  • Google Sheets → Google Sheets Trigger (Row Added)
  • Any database → Postgres / MySQL / MongoDB / Supabase Trigger
What matters: the trigger output must contain a LinkedIn URL field. If your field name is different, you'll need to adjust the next node's expression accordingly.
2
Connect the Bright Data node
The second node is Scrape LinkedIn Profile. Click it and connect your Bright Data credentials (API key from your Bright Data settings).

Dataset is pre-set to LinkedIn people profiles (gd_l1viktl72bvl7bjuj0) - don't touch it.

The URLs field uses this expression:
=[{"url": "{{ $json.fields["LinkedIn URL"] }}"}]
If you swapped the CRM trigger, replace $json.fields["LinkedIn URL"] with the correct path for your trigger (e.g. $json.properties.linkedin_url for HubSpot, or $json["LinkedIn URL"] for Google Sheets).
3
Connect OpenAI (already configured)
The third node is Extract Structured Data. Click it and connect your OpenAI credentials (API key from OpenAI platform).

Model is gpt-5-mini with JSON Object response format. The prompt and schema are already set - nothing to change here unless you want to add/remove fields.

Want a different LLM? You can swap this node for Claude, Gemini, Mistral, or any other model supported by n8n. Keep the system prompt identical.
4
Update the CRM record (last node) - any CRM works
The fourth node is Update Airtable Record. This writes the enriched data back to your CRM. This is the node you need to customize most.

If you kept Airtable: click the node, connect your account, and set your Base ID and Table ID. The template expects these columns to exist:
  • Status, Job Title, Current Company, Seniority Level
  • Years Experience, Industry, Location City, Location Country
  • Tech Stack, Top Certifications, AI Summary, Enriched At
If you swapped the CRM: delete this node and replace it with the equivalent Update node for your CRM. The mappings from the AI output stay the same:
  • Job Title → {{ $json.output[0].content[0].text.current_title }}
  • Company → current_company
  • Seniority → seniority_level
  • Industry, City, Country → matching fields
  • Tech Stack → tech_stack.join(", ")
  • Summary → one_line_summary
Make sure the record ID comes from your trigger (not hardcoded) so the right record gets updated.
5
Set up the Slack notification
The last node is Notify Slack. Connect your Slack account (OAuth2) and pick the channel where you want the alerts (e.g. #leads or #sales). The message template is already pre-formatted with the enriched lead info - no action needed.

Want a different notification channel? Swap the Slack node for Discord, Microsoft Teams, Telegram, Gmail, SMS, or any messaging service supported by n8n.
6
Test and activate
Click Test workflow. Add a test lead in your CRM with a valid LinkedIn URL. Verify that each node turns green and that the CRM record gets enriched + the Slack message arrives. Toggle the workflow to Active. The trigger polls every minute by default.

Troubleshooting

ProblemSolution
Credentials errorCheck Bright Data, OpenAI, CRM, and Slack credentials in each node.
LinkedIn URL expression failsAdjust $json.fields["LinkedIn URL"] to match the field name in your CRM trigger (e.g. HubSpot uses $json.properties.linkedin_url).
OpenAI returns invalid JSONMake sure the response format is set to JSON Object in the OpenAI node settings.
Trigger not firingReduce the polling interval or switch to a webhook-based trigger if your CRM supports it.
OpenAI rate limitReduce trigger frequency or add a Wait node before OpenAI for high-volume use.
