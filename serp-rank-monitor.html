Multi-Country SERP Rank Monitor

Track where you rank on Google, in any country, on autopilot. List your keywords, target countries, and domain in a Google Sheet. On a schedule, Bright Data's SERP API pulls the real local results for each row, finds your domain's position, and writes it back. One sheet, every market, always current.

Zapier template coming soon. Step-by-step guide in progress.
What this scenario does: On a schedule, it reads your keyword list from a Google Sheet (keyword + target country + target domain per row). For each row, Bright Data's SERP API fetches Google's real results in that country, the response is parsed, and the scenario walks the organic results to find where your domain ranks - then writes that position back to the row.

5 modules: Google Sheets (Filter Rows) → Bright Data (SERP API) → Parse JSON → Iterator (organic results) → Google Sheets (Update Row)

Get Make template
Why Bright Data for this: Google shows different results per country, and personalizes / rate-limits scrapers hard. Bright Data's SERP API returns clean, geo-accurate, structured results (an organic array with each result's rank and link) - so you get the real local ranking, not what your own browser sees.
1
Activate the SERP API on Bright Data
This template uses Bright Data's SERP API zone (not Web Unlocker).
  • Log in at brightdata.com
  • Sidebar → Proxies & Scraping Infrastructure → Add → SERP API
  • Name the zone serp_api1 (the name this blueprint expects) and save
  • Grab your API token at brightdata.com/cp/setting/users
Docs: SERP API introduction.
2
Set up your Google Sheet
One row = one keyword you want to track in one country. Create a sheet (Sheet1) with:
ColHeaderExample
AKeywordweb scraping api
BTarget Countryus, fr, de, gb… (Google gl code)
CTarget Domainbrightdata.com
DLast Positionwritten by the scenario
Want the same keyword in 5 countries? Add 5 rows with the same keyword and different country codes - that's the "multi-country" part.
3
Module 1 - Google Sheets (read the keyword list)
The first module reads your rows. Pre-configured:
  • Spreadsheet → point to your sheet · Sheet → Sheet1
  • Table contains headers → Yes · Limit → 10 (rows per run)
Connect your Google account. Each row flows through the next modules one at a time.
4
Module 2 - Bright Data SERP API (fetch local results)
The Bright Data module posts a search request to the SERP API. Pre-configured:
  • Method → POST · URL → /request · Header → Content-Type: application/json
{ "zone": "serp_api1", "url": "https://www.google.com/search?q={{keyword}}&gl={{country}}&hl=en&brd_json=1", "format": "raw" }
Two things do the magic: gl={{country}} pulls results as if searching from that country (column B), and &brd_json=1 returns the SERP as structured JSON - including an organic array where each result has a rank and a link. Connect your Bright Data account.
5
Module 3 - Parse JSON
The SERP API returns a JSON string. This Parse JSON module turns it into structured data so the next module can loop over the organic results. No setup needed.
6
Module 4 - Iterator (walk the organic results)
An Iterator (Basic Feeder) fans out the organic array - one bundle per search result. Each result carries its rank (its position on the page) and its link. This lets the next module check every result against your target domain.
7
Module 5 - Google Sheets (write the rank back)
The Update Row module writes the position back to the same row - but only for the result that matches your domain, thanks to a filter:
  • Filter "Domain match" → only continue when the result's link contains your Target Domain (column C)
  • D Last Position → the matched result's rank
So out of ~10 organic results, only the one that's your domain passes the filter, and its rank lands in column D. If your domain isn't in the results, nothing is written (you're not ranking on page 1 for that country).
8
Schedule, test, and activate
Add a few keyword rows and click Run once - watch column D fill with positions. Then set the scenario's schedule (daily or weekly is typical for rank tracking) and toggle it ON.

Tip: to keep history instead of overwriting, add a dated column each run or append to a log sheet - then you can chart position over time per country.

Troubleshooting

ProblemSolution
No organic array in the responseMake sure &brd_json=1 is in the URL and the zone is a SERP API zone (not Web Unlocker). Without brd_json you get raw HTML.
Position column stays emptyThat usually means your domain isn't on page 1 for that keyword/country. Widen results by requesting more (add &num=20 to the URL) or accept it as "not ranking".
Wrong country's resultsCheck column B uses a valid Google gl code (us, fr, de, gb, es…), lowercase.
Matches a subdomain / wrong pageThe filter is a "contains" on the link. Tighten your Target Domain (e.g. brightdata.com/products) if you only want a specific path.
Only 10 rows processedRaise the Limit on Module 1. Mind your SERP API request volume.
Want alerts on ranking dropsAdd a Router + filter after the update: if new position > last position by N, send a Slack / email alert.
What this workflow does: On a schedule, n8n reads your tracked keywords from a Google Sheet, queries Bright Data for each keyword's Google results, a Code node finds your target domain's position in the organic results, writes the new rank back to the sheet (keeping the previous one), and posts a Slack update.

6 nodes: Schedule → Read Keywords → Bright Data (Web Search) → Code (find rank) → Update Sheet → Slack

Get n8n template
Richer than the Make version: this workflow keeps a Previous Position column (so you see movement, not just the current rank), returns Not in top 100 when you're not ranking, and posts a Slack alert on every update.
1
Set up Bright Data
The Bright Data node uses its Web Search operation to pull Google results.
  • Log in at brightdata.com and create a zone named n8n_unlocker (the name this template expects)
  • Grab your API token at brightdata.com/cp/setting/users
You'll paste the token when you connect the Bright Data node.
2
Set up your Google Sheet
One row per keyword you track. Columns:
ColumnFilled by
Keywordyou (e.g. web scraping api)
Target Countryyou (us, fr, de… — see tip in step 5)
Target Domainyou (e.g. brightdata.com)
Last Positionthe workflow (current rank)
Previous Positionthe workflow (the rank from the run before)
3
Node 1 - Schedule Trigger (weekly, Monday 8am)
The template ships set to run every Monday at 8am - a clean weekly rank snapshot. Click the node to change the cadence (daily, monthly, etc.).
4
Node 2 - Read Keywords from Sheets
Reads all keyword rows from your sheet. Connect your Google account and point documentId to your sheet (replace YOUR_GOOGLE_SHEET_ID), sheet → Sheet1. Each row flows through the next nodes.
5
Node 3 - Fetch SERP Data (Bright Data Web Search)
The Bright Data node runs a Google search per row. Pre-configured:
  • Operation → Web Search · Zone → n8n_unlocker
  • Query → {{ $json.Keyword }} (from the row)
  • Country → us
Make it truly multi-country: the node ships with Country = us. To respect each row's market, set the Country field to an expression → {{ $json['Target Country'] }}. Now every keyword is checked in its own country.
6
Node 4 - Calculate Domain Rank Position (Code)
A Code node walks the organic results and finds where your domain sits:
  • Loops the results, matches the first whose link contains your Target Domain, and takes its rank
  • If the domain isn't found → Not in top 100
  • Carries the sheet's old Last Position into Previous Position, so you keep history
No setup needed - this is the logic that turns a raw SERP into a single position number.
7
Node 5 - Update Rank in Sheet
Writes the result back to the correct row, matched on row_number: Last Position (new rank) and Previous Position (prior rank). Keyword, Country, and Domain are passed through unchanged.
8
Node 6 - Post Update to Slack
Sends a formatted rank update to your channel. Connect Slack and pick the channel. The message:
🔔 SERP Rank Update 🌐 Domain: {{ Target Domain }} 🔍 Keyword: "{{ Keyword }}" 🌍 Country: {{ Target Country }} 📊 Position: {{ Last Position }} (was: {{ Previous Position }})
Prefer another channel? Swap Slack for Discord, Teams, Telegram, Gmail, or a webhook. Or add an IF before it to alert only when the position dropped.
9
Test and activate
Add a few keyword rows and click Test workflow. Check the sheet fills with Last / Previous Position and that the Slack message arrives. When it works, toggle the workflow to Active - it now tracks your rankings on schedule and pings you every run.

Troubleshooting

ProblemSolution
Every row shows the same country's rankThe Web Search node has Country hardcoded to us. Map it to {{ $json['Target Country'] }} (step 5 tip) for real per-market tracking.
Always "Not in top 100"Either you genuinely don't rank, or the domain string is too specific. Try just the bare domain (brightdata.com, no https:// or path).
Code node errors on organicIt reads organic or body.organic. If Bright Data returns a different shape, log one item and adjust the path.
Wrong row updatedThe update matches on row_number. Don't sort/reorder the sheet mid-run, or add a stable unique ID column to match on instead.
Too many Slack messagesAdd an IF after the Code node to only post when Last Position differs from Previous Position.
Want change arrows (▲▼)In the Code node, compare new vs previous and add an emoji to the Slack text based on the delta.
