Google Shopping Price Monitor

Track the lowest price for any product across Google Shopping, on autopilot. List your products in a Google Sheet, and on a schedule Bright Data pulls the Shopping results, Claude filters out accessories and finds the genuine lowest price, and the sheet is updated with the price, the retailer, and a timestamp.

Zapier template coming soon. Step-by-step guide in progress.
What this scenario does: On a schedule, it reads your product list from a Google Sheet. For each product, Bright Data queries its Google Shopping dataset (async: it returns a snapshot, the scenario waits, then pulls the results). Claude reads the results, throws out accessories (cases, cables, chargers…) and picks the genuine lowest price, and the row is updated with the price, retailer, and timestamp.

8 modules: Google Sheets (Read) → Bright Data (Filter Dataset) → Sleep → Get Snapshot Metadata → Get Snapshot Content → Claude (Find Lowest Price) → Parse JSON → Google Sheets (Update Row)

Get Make template
Why Claude is in the loop: a raw Google Shopping search for "AirPods Pro" returns the product plus dozens of cases, tips, and cables - often cheaper than the product itself. A naive "min price" would report the $5 case. Claude keeps only the real product before picking the lowest price, so the number you track is honest.

3
Module 1 - Google Sheets (read the product list)
Reads your product rows. Pre-configured: point it to your spreadsheet, sheet → Sheet1, headers → Yes, limit → 10. Connect your Google account. Each row flows through the rest one at a time.
4
Module 2 - Bright Data (query the Shopping dataset)
The Filter Dataset module searches the Google Shopping products dataset for your keyword. Pre-configured:
  • Dataset → gd_m31f2k0d2m1bah4f3b (Google Shopping products)
  • Filter → title includes {{Product Keyword}}
  • Records limit → 10
This is async: it kicks off the query and returns a snapshot_id (not the data yet). Connect your Bright Data account.
5
Module 3 - Sleep (let the snapshot build)
A Sleep module pauses 300 seconds to give Bright Data time to prepare the snapshot before you ask for it. No setup needed - lower it if your snapshots are consistently ready sooner.
6
Module 4 - Get Snapshot Metadata (check status)
The Get Snapshot Metadata module checks the snapshot from Module 2 and returns its status. When the query is finished, status becomes ready. Pre-mapped to the snapshot_id.
7
Module 5 - Get Snapshot Content (pull the products)
The Get Snapshot Content module downloads the actual product records as JSON.
  • Filter → only runs when status = ready (so it won't grab a half-built snapshot)
  • Format → json
If a snapshot isn't ready in time, this step is skipped for that row - increase the Sleep, or add a retry loop.
8
Module 6 - Claude (filter accessories, find lowest price)
This is what makes the number trustworthy. The Claude (Opus 4.8) module gets the keyword + the Shopping results and is told to:
  • Keep only the genuine product (exclude cases, ear tips, cables, chargers, covers, replacement parts)
  • Among those, find the lowest price
It returns strict JSON:
{ "lowest_price": <number>, "retailer": "<tag>", "title": "<title>", "product_count_valid": <number> }
Connect your Anthropic credentials. Tweak the exclusion list in the prompt for your category (e.g. for a laptop: "exclude sleeves, chargers, docks").
9
Module 7 - Parse JSON
Turns Claude's reply into clean fields - lowest_price, retailer, title - ready to write to the sheet. No setup needed.
10
Module 8 - Google Sheets (write the price back)
Updates the same row (matched by row number):
  • A → the matched product title · B → lowest price · C → retailer · D → {{now}} timestamp
Want price history? Instead of overwriting, append a new dated row to a log sheet so you can chart price over time.
11
Schedule, test, and activate
Add a few products and click Run once (remember the ~5-min Sleep, so give it time). Check columns B-D fill in. Then set the scenario's schedule (daily is typical for price monitoring) and toggle it ON.

Add a drop alert: after the update, compare the new price to the old one and fire a Slack / email alert when it falls below your target.

Troubleshooting

ProblemSolution
Snapshot content is emptyThe snapshot wasn't ready when Module 5 ran. Increase the Sleep (300s → higher) or add a loop that re-checks metadata until status = ready.
Lowest price looks too lowAn accessory slipped through. Tighten the exclusion list in Claude's prompt for your product category.
No matching productsThe title includes filter is strict. Use a broader keyword, or the core model name without extra words.
Claude JSON won't parseThe prompt forces strict JSON with no code fences. If a model adds prose, add "Output ONLY the JSON object" (already in the prompt) or strip fences before Parse JSON.
Only 10 products processedRaise the Limit on Module 1. Mind your Bright Data dataset usage.
Want a different country/currencyUse the matching regional Google Shopping dataset, or filter on currency in Module 2.

Get n8n template
Smarter than the Make version: instead of one fixed 5-minute Sleep, this workflow polls in a loop (Wait 50s → check status → if not ready, loop back). It moves on the moment the snapshot is ready - faster on quick jobs, and it won't fail on slow ones.

  • Filter → title includes {{ $json["Product Keyword"] }}

6
Nodes 4-6 - Poll until ready (Wait → Check → IF)
This is the reliable part. Three nodes form a polling loop:
  • Wait 50 Seconds - give the snapshot time to build
  • Verify Snapshot Completion - getSnapshotMetadata returns the status
  • If Snapshot Ready - if status = ready → continue; otherwise → loop back to Wait and check again
So it keeps checking every 50s until Bright Data is done - no guessing, no fixed delay. Increase the Wait if you monitor many products at once.

{ "lowest_price": <number>, "retailer": "<tag>", "title": "<title>", "product_count_valid": <number> }
Connect your Anthropic credentials. Tweak the exclusion list in the prompt for your category.

A Set node parses Claude's JSON into clean columns - Lowest Price, Retailer, Product Title, Last Checked (today's date) - and carries the row_number so the right row gets updated.

Want price history? Swap the update for an append to a log sheet so each run adds a dated row - then chart price over time.
