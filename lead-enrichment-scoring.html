LinkedIn Lead Enrichment & AI Scoring

Drop a LinkedIn URL into a Google Sheet and the rest is automatic. Bright Data scrapes the full profile, Claude scores the lead 0-100 with a one-line reason, and the enriched data plus the score are written straight back to the row. Your sheet becomes a self-updating, ranked lead list.

Zapier template coming soon. Step-by-step guide in progress.
What this scenario does: A new row in your Google Sheet (a LinkedIn URL is all you need) triggers Bright Data to scrape the full LinkedIn profile via its structured dataset. The JSON is parsed, Claude scores the lead 0-100 with a one-sentence reason, and the row is updated with the enriched fields, the score, and a DONE status.

6 modules: Google Sheets (Watch Rows) → Bright Data (Scrape Profile) → Parse JSON → Claude (Score Lead) → Parse JSON → Google Sheets (Update Row)

Get Make template
Source and destination are swappable: the scenario reads and writes a Google Sheet by default, but the same flow works with Airtable, Notion, or a CRM (HubSpot, Pipedrive, Attio) as the lead source. Swap the first and last modules and keep the Bright Data → Claude core untouched.
1
Get your Bright Data API token
This template scrapes LinkedIn profiles through Bright Data's structured dataset API (the LinkedIn People Profiles dataset) - you get clean fields like title, company, location, and connections, not raw HTML.
  • Log in at brightdata.com
  • Go to Settings → API tokens and copy your token (brightdata.com/cp/setting/users)
  • You'll paste it when you connect the Bright Data module in step 4
No zone setup needed - the dataset API handles the scraping.
2
Set up your Google Sheet
The scenario reads new rows and writes results back to the same row. Create a sheet (Sheet1) with these columns:

You only ever fill column B (and optionally A). Everything else is written automatically.
3
Module 1 - Google Sheets (Watch Rows)
The trigger watches your sheet for new rows. Pre-configured:
  • Spreadsheet → point it to your sheet · Sheet → Sheet1
  • Table contains headers → Yes · Limit → 3 (rows processed per run)
Connect your Google account. Make polls on the scenario's schedule, so each new LinkedIn URL you paste gets picked up on the next run.
4
Module 2 - Bright Data (scrape the LinkedIn profile)
The Bright Data module calls the dataset API and passes the LinkedIn URL from column B. Pre-configured:
  • URL → /datasets/v3/scrape?dataset_id=gd_l1viktl72bvl7bjuj0&include_errors=true
  • Body → [{"url":"{{2.LinkedIn URL}}"}] (column B from the trigger)
  • Header → Content-Type: application/json
gd_l1viktl72bvl7bjuj0 is Bright Data's LinkedIn People Profiles dataset. Connect your Bright Data account (the token from step 1). It returns the profile as structured JSON.
5
Module 3 - Parse JSON (the profile)
Bright Data returns the profile as a JSON string. This Parse JSON module turns it into usable fields - name, current_company.title, current_company.name, location, connections, followers, and more - so the next modules can map them.

No setup needed; the data structure is already defined in the template.
6
Module 4 - Claude (score the lead)
This is the brain of the scenario. The Anthropic Claude (Opus 4.8) module reads the enriched profile and scores the lead. Pre-configured:
  • Model → claude-opus-4-8 · Max tokens → 1000
  • Scores 0-100 on seniority, company relevance, and decision-making power for a B2B SaaS sales team
  • Returns strict JSON: {"score": <0-100>, "reason": "<one sentence>"}
Connect your Anthropic credentials. Tune the scoring by editing the prompt - change the ICP ("score for an agency selling to e-commerce"), add weighting rules, or ask for a tier (A/B/C) instead of a number.
7
Module 5 - Parse JSON (the score)
A second Parse JSON module turns Claude's reply into two clean fields - score and reason - ready to write to the sheet. No setup needed.
8
Module 6 - Google Sheets (Update Row)
Writes everything back to the same row (matched by row number from the trigger):
  • C Title, D Company, E Location, F Connections → from the parsed profile
  • G Lead Score → Claude's score
  • H Status → DONE (so the row won't be processed again)
If you swapped the destination (Airtable / Notion / CRM): replace this module with the equivalent Update action and map the same fields.
9
Test, schedule, and activate
Paste one real LinkedIn URL into column B and click Run once. Watch the row fill in with title, company, location, connections, a score, and DONE.

Once it works, set the scenario's schedule (e.g. every 15 min) and toggle it ON. Now anyone can drop LinkedIn URLs into the sheet and get back a ranked, enriched lead list. Sort by column G to work your hottest leads first.

Troubleshooting

ProblemSolution
Bright Data returns empty / errorCheck the LinkedIn URL is a full profile URL (linkedin.com/in/...). The include_errors=true flag surfaces the reason in the response.
Same rows processed twiceThe Status = DONE write prevents this. If it still loops, add a filter on the trigger to skip rows where column H is already DONE.
Claude output won't parseThe prompt forces strict JSON. If a model adds prose, lower the temperature or add "Output ONLY the JSON object" - already in the prompt.
Score column shows the reason tooMake sure column G maps to {{score}} from Module 5, not the whole Claude result.
Want to scrape more per runRaise the Limit on the Watch Rows trigger (default 3). Mind your Bright Data and Anthropic rate limits.
Need company-level data tooAdd a second Bright Data module using the LinkedIn Company dataset, keyed on current_company.company_id.
What this workflow does: When a new row lands in your Google Sheet, n8n reads the leads and loops through them one by one. For each, Bright Data scrapes the person's LinkedIn profile and their company, Claude scores the lead 0-100 against your ICP, and the row is updated - leads scoring ≥ 70 are marked HOT 🔥, the rest DONE.

11 nodes: Sheets Trigger → Read Leads → Loop → Bright Data (Profile) → Wait → Bright Data (Company) → Wait → Claude (Score) → Parse JSON → IF (≥70?) → Update Sheet (HOT / DONE)

Get n8n template
Richer than the Make version: this workflow also scrapes the company (size + industry), adds a Reason column, and routes hot leads to a HOT 🔥 status so your team works the best ones first. It has wait steps to respect Bright Data / LinkedIn rate limits.
1
Get your Bright Data API token (two datasets)
This workflow uses two of Bright Data's structured LinkedIn datasets:
  • LinkedIn People Profiles → gd_l1viktl72bvl7bjuj0
  • LinkedIn Company Information → gd_l1vikfnt1wgvvqz95w
Grab your API token at brightdata.com/cp/setting/users - you'll paste it when you connect the Bright Data nodes.
2
Set up your Google Sheet
Create a sheet (Sheet1) with these columns. You only fill Name + Linkedin URL; the workflow writes the rest:
ColumnFilled by

3
Node 1 - Google Sheets Trigger (new row added)
The Google Sheets Trigger fires when a row is added, polling every minute. Connect your Google account and point documentId to your sheet (replace YOUR_GOOGLE_SHEET_ID), sheet → Sheet1.

Prefer a schedule instead of live polling? Swap for a Schedule Trigger and keep the read node below.
4
Node 2 - Read Leads from Sheets
Reads the lead rows from the sheet. A filter on the Status column lets you process only un-scored rows.

Tip: set the filter to pull rows where Status is empty, so already-scored leads (HOT / DONE) are skipped.
5
Node 3 - Batch Process Leads (Loop)
The Loop (Split In Batches) node processes one lead at a time. Both update branches loop back here, so it keeps going until every lead is scored. Keeping it one-at-a-time is what makes the rate-limit waits below work. No setup needed.
6
Node 4 - Scrape LinkedIn Profile (Bright Data)
The Bright Data node scrapes the person's profile using the LinkedIn People Profiles dataset. Pre-configured:
  • Resource → Web Scraper · Dataset → gd_l1viktl72bvl7bjuj0
  • URLs → [{"url":"{{ $json["Linkedin URL"] }}"}] from the current row
Returns clean fields: name, current_company.title, city, connections, and the experience array (used next for the company).
7
Node 5 - Wait 30 Seconds
A Wait node pauses 30s between the two Bright Data scrapes to respect rate limits. Adjust if you're on a higher Bright Data tier.
8
Node 6 - Scrape LinkedIn Company (Bright Data)
A second Bright Data node enriches the company using the LinkedIn Company Information dataset. Pre-configured:
  • Dataset → gd_l1vikfnt1wgvvqz95w
  • URLs → [{"url":"{{ $json.experience[0].url }}"}] - the company from the profile's first experience
Returns name, company_size, industries - the firmographics Claude uses for a sharper score.
9
Node 7 - Pause for 30 Seconds
A second Wait node before the AI call, again for rate limits. No setup needed.
10
Node 8 - AI Analyze Lead Score (Claude)
This is the brain - and the one node you must edit. The Claude (Opus 4.8) node scores the lead against your ICP. The prompt has placeholders to fill in:
WE SELL: [your product] OUR IDEAL CUSTOMER: [e.g. founders, heads of sales / growth / RevOps at B2B SaaS companies, 10-500 employees, doing outbound]
It feeds Claude the person (name, title, location, connections) + company (name, size, industry) and asks for strict JSON: {"score": <0-100>, "reason": "<one sentence>"}. Connect your Anthropic credentials and replace the bracketed lines with your real offer and ICP.
11
Node 9 - Parse Lead Score JSON (Code)
A small Code node strips any markdown fences from Claude's reply and parses it into score and reason. No setup needed.
12
Node 10 - Check If Score ≥ 70 (IF)
The IF node splits the flow on the score: ≥ 70 → hot lead branch, otherwise → done branch.

This is your hot-lead threshold - change 70 to match how selective your team wants to be.
13
Node 11 - Update the Sheet (HOT 🔥 / DONE)
Two Google Sheets - Update nodes write the result back to the row and then loop back to the batch:
  • Hot branch → Status = HOT 🔥
  • Done branch → Status = DONE
Both write Title, Company, Company Size, Location, Connections, Lead Score, Reason. Matched on the row so the right lead gets updated.
14
Test and activate
Add one real LinkedIn URL to the sheet and click Test workflow. Mind the two 30s waits - a single lead takes ~1 min. Check the row fills with profile + company data, a score, a reason, and HOT 🔥 or DONE.

When it works, toggle the workflow to Active. New rows now get scored automatically. Filter your sheet on HOT 🔥 to see your best leads.

Troubleshooting

ProblemSolution
Company node returns nothingIt reads experience[0].url from the profile. If the lead has no listed experience, that's empty - add an IF to skip the company scrape when there's no company URL.
Rate-limit / 429 errorsIncrease the two Wait nodes (30s → 60s) or lower the batch size so fewer leads hit Bright Data per minute.
Claude JSON won't parseThe Code node already strips ```json fences. If it still fails, log the raw text and tighten the prompt's "ONLY valid JSON" line.
Every lead scores lowYou didn't fill the ICP placeholders in Node 8 - Claude is scoring against a generic profile. Put your real product + ideal customer in the prompt.
Same leads re-processedFilter the Read node to rows where Status is empty, so HOT / DONE rows are skipped.
Want Slack alerts on hot leadsAdd a Slack node on the hot branch before it loops back, posting the name, score, and reason.
