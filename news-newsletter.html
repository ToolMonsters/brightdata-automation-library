Automated News Newsletter

On a schedule, Bright Data searches the web for news on any topic, picks the top five stories, then scrapes and cleans the full article text of each one. Claude writes a polished, branded newsletter from the aggregated content and it goes out to your list via Beehiiv, ConvertKit, Mailchimp, or Gmail.

Zapier template coming soon. Step-by-step guide in progress.
What this scenario does: Bright Data runs a news search across the web for your topic and returns the results as structured JSON. That JSON goes straight to Claude, which picks the top 5 stories and writes the newsletter as ready-to-send HTML. It goes out to your list via Beehiiv, ConvertKit, Mailchimp, or Gmail.

3 modules: Bright Data (Web Unlocker) → Claude (Write Newsletter) → Email (Beehiiv / ConvertKit / Mailchimp / Gmail)

Get Make template
Simpler than the n8n version: this scenario summarizes straight from the news search result (title + snippet + link), so it skips the per-article scraping loop. It's faster and cheaper. Want full-article summaries? Add a Bright Data module between steps 1 and 2 to fetch each link, or use the n8n template which already loops.

1
Activate Web Unlocker on Bright Data
This is what makes the news search reliable - Web Unlocker handles anti-bot, JavaScript, and geo-blocks automatically, on the search results and on any news site you land on.
  • Log in at brightdata.com
  • Sidebar → Proxies & Scraping Infrastructure
  • Click Add → select Web Unlocker
  • Click Start with Web Unlocker or Create zone
  • Name it n8n_serp (the name this blueprint expects) and save
Full docs: Web Unlocker introduction.
2
Module 1 - Bright Data Web Unlocker (search the web for news)
The first module runs a news search across the web via Bright Data and returns the parsed results. Pre-configured:
  • Zone → n8n_serp (the zone from step 1)
  • URL → https://www.google.com/search?q=AI+automation&tbm=nws&gl=us&hl=en&brd_json=1
  • Format → Raw  ·  Method → GET  ·  Async → off
The trick is &brd_json=1 at the end of the URL - it makes Bright Data parse the page and return a news array (titles, links, sources) you can reference directly as {{2.result.news}}, no parser needed.

Connect your Bright Data account. To change the topic, edit the q= part of the URL (e.g. q=fintech+funding). The tbm=nws param scopes the search to news results.
3
Module 2 - Claude (write the newsletter)
This is where the brief gets written. The Anthropic Claude (Opus 4.8) module receives the news JSON and writes the newsletter as clean HTML. Pre-configured:
  • Model → claude-opus-4-8
  • Max tokens → 2000
  • Prompt → includes {{2.result.news}} and asks for the top 5 stories only
For each story Claude returns a bold headline, a 1-2 sentence summary in its own words, and a "Read more" link - plus a one-line intro. Output is pure HTML, no markdown fences.

Connect your Anthropic credentials. Tune the voice by editing the prompt. Want a different LLM? Swap for OpenAI or another provider module.
4
Module 3 - Send the newsletter (Beehiiv / ConvertKit / Mailchimp / Gmail)
The blueprint ships with Gmail - Send an Email so it runs out of the box. Connect your account and set:
  • To → your recipient(s)
  • Subject → e.g. Your AI Industry Brief
  • Content → map to Claude's output {{4.result}}, and set the content type to HTML
Sending to a real audience? Swap the Gmail module for your newsletter platform and map the same HTML body:
  • Beehiiv - create a post / send a broadcast via its API
  • ConvertKit - create a broadcast
  • Mailchimp - create and send a campaign
  • Brevo / Outlook / SMTP - any email module
You can also post the HTML to Slack / Notion / a webhook instead of emailing.
5
Schedule, test, and activate
Click Run once and send yourself a test email first to check the formatting. When you're happy, open the scenario's schedule settings (clock icon on the first module) to set the cadence - daily, weekly, etc. - and toggle the scenario ON.

Troubleshooting

ProblemSolution
{{2.result.news}} is emptyMake sure &brd_json=1 is still at the end of the URL and tbm=nws is present. Without brd_json you get raw HTML, not the parsed array.
Claude output has markdown fencesThe prompt says HTML only. If a model adds ```html, add a small Text Parser / Set Variable to strip fences before Gmail.
Email body shows raw HTML tagsSet the Gmail content type to HTML, not plain text.
Newsletter too long / too shortAdjust Max tokens on the Claude module, or tell the prompt how many stories and how many sentences each.
Want full-article summariesInsert a Bright Data module that fetches each link (format markdown) before Claude, or use the n8n template which already loops over articles.
What this workflow does: On a schedule, n8n uses Bright Data to search the web for news on your topic, a Code node filters out videos and duplicates and keeps the top 5 stories, then loops through each one to scrape the full article via Bright Data and clean it into plain text. After the loop, all articles are aggregated and sent to Claude, which writes the newsletter as ready-to-send HTML. It goes out to your list via Beehiiv, ConvertKit, Mailchimp, or Gmail.

9 nodes: Schedule → Bright Data (News Search) → Select Top 5 (Code) → Loop → Bright Data (Article) → Sanitize (Code) → Aggregate → Claude (Write Newsletter) → Email

2
Node 1 - Schedule Trigger (set your frequency)
The template ships with a Schedule Trigger set to fire every Monday (a weekly brief). Click the node to change the cadence - daily, twice a week, monthly, whatever fits your audience.

Want a different trigger? Swap for a Webhook (send on demand), Form Trigger (generate a brief by submitting a topic), or Chat Trigger.
3
Node 2 - Search the web for news with Bright Data
The Bright Data node runs a news search across the web and returns the results as clean structured JSON. Pre-configured:
  • Zone → n8n_unlocker (the zone from step 1)
  • Country → us (change for a different geo)
  • URL → https://www.google.com/search?q=AI+automation&tbm=nws&gl=us&hl=en&brd_json=1
The trick is &brd_json=1 at the end of the URL - it tells Bright Data to parse the page and return a news array as JSON instead of raw HTML, so you get titles, links, and sources without writing a parser.

To change the topic, just edit the q= part of the URL (e.g. q=fintech+funding, q=climate+tech). The tbm=nws param is what scopes the search to news results.
4
Node 3 - Select Top 5 News Stories (Code)
A Code node takes the raw news array and cleans it down to the stories worth covering:
  • Drops YouTube / video links (keeps real articles only)
  • Removes entries missing a title or link
  • De-duplicates by title
  • Keeps the top 5 and emits each as a separate item for the loop
Want a longer brief? Change .slice(0, 5) to 10 (or any number). No other setup needed.
5
Node 4 - Loop Over News Stories (Split In Batches)
The Loop node processes one story at a time. Each pass scrapes that story's full article (next two nodes), then control returns to the loop. When every story is done, the loop's "done" output continues to the aggregate + newsletter steps.

This keeps things resilient - if one article fails to scrape, the rest still make it into the newsletter. No setup needed.
6
Node 5 - Fetch Article Content with Bright Data
For each story in the loop, the Bright Data node opens the article link and returns the page. Pre-configured:
  • Zone → n8n_unlocker
  • URL → {{ $json.link }} from the current story
  • Format → markdown (clean text, no HTML soup)
Returning markdown means most of the page chrome is already stripped before it reaches the cleanup node. Web Unlocker handles paywalls, cookie banners, and JS rendering automatically.
7
Node 6 - Sanitize Article Content (Code)
A Code node turns the raw markdown into clean newsletter-ready text:
  • Handles whatever shape Bright Data returns (array, string, or object)
  • Jumps to the last # heading to skip nav and headers
  • Strips markdown links and images, keeping the visible text
  • Keeps only real sentences (lines > 50 chars) and caps it at 2,500 chars
  • Re-attaches the title, link, and source from the loop
No setup needed. Raise the 2500 cap if you want Claude to see more of each article.
8
Node 7 - Compile All Articles (Aggregate)
Once the loop finishes, the Aggregate node collects every cleaned article into a single item (one data array) so Claude can see all five stories at once. No setup needed.
9
Node 8 - Compose Newsletter with Claude
This is where the brief gets written. The Anthropic Claude (Opus 4.8) node receives all five cleaned articles and writes the newsletter as clean HTML, ready to drop into an email.

The system prompt tells Claude to produce, for each story: a bold headline, a 2-3 sentence summary in its own words (not copied), and a "Read more" link - plus a one-line intro at the top. Output is pure HTML, no markdown fences.

Connect your Anthropic credentials. Tune the voice by editing the system prompt (tone, length, sections, add an emoji header, etc.). Want a different LLM? Swap for OpenAI GPT-5, Gemini, or a local model.
10
Node 9 - Send the Newsletter (Beehiiv / ConvertKit / Mailchimp / Gmail)
The template ships with a Gmail node so it runs out of the box - pre-configured with a subject (Your AI Industry Brief) and the body mapped to Claude's output.
  • Connect your Gmail account, set To, Subject, and confirm the body maps to {{ $json.content[0].text }}
Sending to a real audience? Swap the Gmail node for your newsletter platform and map the same HTML body:
  • Beehiiv - create a post / send a broadcast (HTTP Request node to its API)
  • ConvertKit - create a broadcast
  • Mailchimp - create and send a campaign
  • Brevo / Resend / Customer.io / Outlook / SMTP - any email node
You can also post it to Slack / Notion / a webhook instead of email.

Troubleshooting

ProblemSolution
News search node returns no news arrayMake sure &brd_json=1 is still at the end of the URL and tbm=nws is present. Without brd_json you get raw HTML, not the parsed array.
Empty or junk article textThe Sanitize node keeps lines > 50 chars. For sites that load body text via JS, the markdown format usually handles it; if not, raise the char cap or relax the line filter.
Fewer than 5 stories in the briefThe dedupe + video filter can thin out results. Broaden the query, or pull more before slicing (e.g. scrape, then .slice(0, 8)).
Claude output has markdown fencesThe prompt says HTML only, but if a model adds ```html, add a tiny Code node to strip fences before Gmail.
Email body shows raw HTML tagsMake sure the Gmail node is sending as HTML, not plain text.
One article kills the runSplit In Batches continues past failures by design - check the execution log to see which link errored.
