New Job Posting as Outbound Sales Signal

Turn LinkedIn job postings into HubSpot deals on autopilot. Bright Data scrapes new hiring signals daily, AI qualifies which postings indicate buying intent, drafts a cold email, and creates a deal in HubSpot with a Slack alert. Works with any CRM and any messaging tool.

Zapier template coming soon. Step-by-step guide in progress.
What this scenario does: On a schedule, the scenario reads a saved LinkedIn jobs search URL from your storage (Airtable by default), uses Bright Data Web Unlocker to fetch the page, and asks GPT to extract every job posting as structured data. Each new job is qualified by AI for buying intent, gets a personalized cold email drafted, lands as a deal in your CRM, and a Slack alert pings your sales team.

Flow: Storage (Searches) → Bright Data (Scrape LinkedIn jobs) → GPT (Parse jobs JSON) → Iterator (loop jobs) → GPT (Qualify + draft email) → CRM (Create Deal) → Slack (Alert)

Get Make template
Storage, CRM, and alerts are swappable: the template uses Airtable for storage, HubSpot for CRM, and Slack for alerts by default. Storage can be Notion, Google Sheets, or any database. CRM can be Pipedrive, Salesforce, Attio, Close. Alerts can go to Discord, Teams, Telegram, Gmail, or any messaging app.
1
Activate Web Unlocker on Bright Data
Web Unlocker fetches LinkedIn job pages reliably (handles JS rendering, bot detection, geo).
  • Log in at brightdata.com
  • Sidebar → Proxies & Scraping Infrastructure
  • Click Add → select Web Unlocker
  • Click Start with Web Unlocker → name it web_unlocker_agent
Full docs: Web Unlocker introduction.
2
Set up your Searches storage - any database works
The storage holds your LinkedIn job search URLs (one per role/region/keyword you want to monitor). Pick what fits:
  • Airtable - template default
  • Notion / Google Sheets / any database (Postgres, Supabase, MongoDB)
Required fields:
  • Label - friendly name (e.g. "AE roles in EU SaaS")
  • LinkedIn URL - the LinkedIn jobs search URL (e.g. linkedin.com/jobs/search?keywords=Account%20Executive&location=Paris&f_TPR=r86400 for past 24h)
  • Active (boolean) - toggle monitoring per search
  • Last Run (date) - filled by the scenario
  • Last Job Count (number) - filled by the scenario
  • Notes - optional
Pro tip: use LinkedIn's filter f_TPR=r86400 (past 24h) so you only get fresh postings on each run. The quality of your output depends entirely on the quality of these URLs - take time to build precise searches per ICP/role/region.
3
Module 1 - Search Records (read active searches)
The first module is Airtable - Search Records. It reads one active search at a time, sorted by Last Run ascending (so the oldest one runs first - good rotation when you have many searches).
  • Filter → {Active} = TRUE()
  • Sort → Last Run ascending
  • Limit → 1 (process one search per run)
If you swapped the storage: replace this module with the equivalent Search/List action.
4
Module 2 - Bright Data Web Unlocker (scrape LinkedIn jobs)
Fetches the LinkedIn jobs search URL from the previous module. Pre-configured:
  • Zone → web_unlocker_agent
  • URL → {{1.LinkedIn URL}}
  • Format → raw
  • Method → GET
Returns the raw HTML of the LinkedIn job results page.
5
Module 3 - GPT (parse jobs into structured JSON)
OpenAI GPT-5-mini reads the LinkedIn HTML and extracts every job posting as a structured JSON array.

For each job: title, company, company_linkedin_slug, location, posted_date, posted_iso, job_url, early_applicant, actively_hiring.

The system prompt is already configured. Limited to 30 jobs per run to stay within the model's context window.
6
Module 4 - Parse JSON
JSON - Parse JSON turns GPT's text response into a real array Make can iterate over. Already pre-configured with the schema (title, company, location, etc.).
7
Module 5 - Iterator (loop over each job)
The Iterator module processes one job at a time so each gets its own qualification, email draft, deal, and alert.
8
Module 6 - GPT (qualify buying intent + draft email)
For each job, a second OpenAI module:
  • Scores buying intent based on the role and seniority (e.g. "Head of Marketing" hiring → likely needs marketing tools)
  • Drafts a personalized cold email that ties the hiring signal to your value prop
Tweak the prompt with your ICP, your product, and the angle you want the email to take. Returns JSON with intent_score, reasoning, email_subject, email_body.
9
Module 7 - Filter (only qualified jobs)
A Make filter on the next module continues only if intent_score >= 7 (or whatever threshold you pick). This avoids creating noise in your CRM for low-signal jobs.
10
Module 8 - Create the Deal in your CRM - any CRM works
HubSpot - Create a Deal by default. The deal includes:
  • Deal name → {{company}} - {{title}}
  • Amount → estimated (or leave blank)
  • Stage → New Lead (or your initial stage)
  • Notes → AI reasoning + drafted email body + LinkedIn job URL
Want a different CRM? Replace this module with:
  • Pipedrive → Create a Deal
  • Salesforce → Create an Opportunity
  • Attio → Create a Record
  • Close → Create a Lead
  • Airtable / Notion → Create a Record (CRM-lite)
11
Module 9 - Send the alert (any channel works)
Slack - Create a Message by default. The alert summarizes the new opportunity:
🎯 New hiring signal: *{{company}}* 🧑‍💼 Role: {{title}} ({{location}}) 📊 Intent score: {{intent_score}}/10 🔗 LinkedIn job: {{job_url}} 📨 Draft email: {{email_body}}
Want a different channel? Discord, Microsoft Teams, Telegram, Gmail, Twilio SMS, or any messaging app supported by Make.
12
Module 10 - Update Last Run on the Search
Updates the Airtable Search record:
  • Last Run → {{now}}
  • Last Job Count → number of jobs found
This way, the next scenario run picks the next search in rotation (sort by Last Run ascending).
13
Test and activate
Add 1-2 LinkedIn job search URLs to your Airtable with Active = TRUE. Click Run once. Verify that jobs are scraped, qualified, and that one HubSpot deal + Slack alert appears for a high-intent job.

Set the schedule (every 1-6 hours depending on your volume) and toggle the scenario ON. The "1 search per run" pattern means each search gets its turn over time without overwhelming your CRM.

Troubleshooting

ProblemSolution
LinkedIn returns 0 jobsMake sure the LinkedIn URL is a valid jobs search (try opening it in incognito). Add f_TPR=r86400 for past 24h to limit to fresh postings.
GPT extracts incomplete dataThe model is limited to 30 jobs per run to stay within context. Use a more granular search URL (specific keyword + location) to keep results focused.
Too many low-quality deals in CRMRaise the intent_score threshold in the filter (try 8 or 9). Or refine the qualification prompt with stricter criteria for your ICP.
Duplicate deals in CRMAdd a search step before Create Deal to check if a deal with the same company + role already exists. Skip if yes.
Cold emails feel genericInject more context into the GPT prompt (your product description, your ICP, sample winning emails). The more context, the more personalized the output.

HIGH and MEDIUM signals create a HubSpot deal and ping Slack with the full context.

  • Zone → n8n_unlocker
  • URL → {{ $json.fields["LinkedIn URL"] }} from the loop
  • Country → us (change if needed)
  • Data format → markdown (clean output for the LLM)
Connect your Bright Data credentials. Web Unlocker handles JS rendering, CAPTCHAs, and bot detection automatically.

Node 5 - Extract Jobs (AI)

For each job: title, company, company_linkedin_slug, location, posted_date, posted_iso, job_url, early_applicant, actively_hiring.

8
Node 6 - Parse JSON Output (Code node)
A small Code node parses the GPT response into a real array. It also handles edge cases: strips markdown code fences if the model wraps the JSON, and recovers as much as possible if the output is truncated mid-job.

No setup needed - just keep the node as-is unless you want to customize the recovery logic.
9
Node 7 - Score Signal & Draft Email (AI)
OpenAI GPT-4o-mini with a strict JSON schema scores each job as a buying signal:
  • HIGH - Senior leadership hire, multiple roles open, net-new sales hire at B2B SaaS
  • MEDIUM - Mid-level B2B sales/eng/marketing roles at growing companies
  • LOW - Junior IC roles, sports teams, retail/hospitality, agency staffing
For HIGH and MEDIUM, GPT also drafts a 3-4 sentence personalized cold email that ties the hiring signal to your value prop. Returns signal_strength, rationale, outreach_email.

Tweak the prompt with your ICP, your product, and your tone - the more context, the more personalized the email.
10
Node 8 - HIGH or MEDIUM Signal? (smart filter)
The IF node only continues if signal_strength = HIGH OR signal_strength = MEDIUM. LOW signals are skipped (no CRM noise, no Slack spam).

Already pre-configured. Adjust the threshold if you want only HIGH signals to go through.

Troubleshooting

ProblemSolution
Web Unlocker 403 / timeoutMake sure your Web Unlocker zone is created and active. Try a different country in the Bright Data node if LinkedIn blocks the geo.
GPT returns truncated JSONThe Parse JSON Output Code node already handles truncation. If you still see issues, lower the job count limit in the Extract Jobs prompt (try 20 instead of 30).
Too many low-quality deals in CRMTighten the IF filter to only allow HIGH signals. Or refine the scoring prompt with stricter criteria for your ICP.
Duplicate deals in HubSpotAdd a HubSpot Search node before Create Deal to check if a deal with the same company + role already exists. Skip if yes.
Cold emails feel genericAdd your product description, ICP, and 1-2 winning email examples to the Score Signal prompt. The more context, the better the output.
Loop fails on one searchBy design, Split In Batches continues even if one item fails. Check the execution log to see which URL errored.
