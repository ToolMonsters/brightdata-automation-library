AI Agent Web Research Loop

Give an AI agent a research goal. It autonomously browses the web using Bright Data Web Unlocker as its web-access tool, iteratively refines its search, and writes a structured report to your destination. Built on the native AI Agent feature of each platform.

Zapier template coming soon. Step-by-step guide in progress.
What this scenario does: You drop a research goal + target audience into a Google Sheet. Make's native AI Agent (Claude Opus 4.7) receives the goal and uses Bright Data Web Unlocker as a connected tool to browse the web autonomously. The agent searches Google, identifies high-quality sources, fetches each page, cross-references findings, and produces a Slack-ready structured report. A final Slack message posts the report to your team.

3 modules: Google Sheets (Watch Rows) → AI Agent (with Bright Data Web Unlocker tool) → Slack (Create Message)

Get Make template
Works with any input and output: the template uses Google Sheets as input and Slack as output by default. Swap the first module for a Webhook, Airtable, Notion, Form, or Slack slash command. Swap the last module for Notion, Airtable, Gmail, Discord, Teams, or any Make-supported app.
1
Set up the trigger (first module) - any source works
The trigger can be anything. Pick whatever matches your use case:
  • Webhook → POST a JSON payload from any app, form, or another scenario
  • Google Sheets - Watch Rows → drop research goals into a sheet
  • Airtable / Notion - Watch Records → process new rows / pages
  • Slack - Watch Slash Command → trigger from /research [topic]
  • Schedule → run a recurring research task
  • Form / Typeform → public intake form
The template ships with Google Sheets as a placeholder (columns: Research goal, Target audience), but feel free to swap it.

What matters: whatever trigger you use must expose a research goal field (and optionally a target audience) that the AI Agent can read in the next step.
2
Configure the AI Agent module (the core of this workflow)
The second module is Make's native AI Agent. This is where the autonomous research happens.

Click the module and connect your LLM provider. The template uses Claude Opus 4.7 by default (best for agentic workflows) but you can swap for GPT-5, Gemini, Mistral, etc.

Configuration already pre-set:
  • Steps per agent call (recursionLimit) → 300
  • Reasoning effort → low (switch to medium/high for harder topics)
  • Response format → Text (Slack markdown)
The system prompt instructs the agent to:
  • Use Google search URLs to discover sources
  • Identify 5-7 high-quality sources (prefer primary sources)
  • Fetch each with the Web Unlocker tool
  • Cross-reference findings, search more if unclear
  • Stop when it has strong evidence from 5+ sources
  • Output a Slack-ready report (Executive Summary, Key Findings, Deep Dive, Sources)
The Input field is mapped to pull the research goal + audience from the trigger.
3
Connect Bright Data Web Unlocker as a tool
The AI Agent has one tool attached: Run a Web Unlocker. This is what makes the agent "see" the web.

First, activate Web Unlocker on Bright Data (if not already done):
  • Log in at brightdata.com
  • In the left sidebar, go to Proxies & Scraping Infrastructure
  • Click Add (or Get started)
  • Select Web Unlocker from the list of zone types
  • Click Start with Web Unlocker or Create zone
  • Name the zone (e.g. web_unlocker_agent) and save
Full docs: Web Unlocker introduction.

Then, in the Make scenario, click the tool sub-flow and connect your Bright Data account (API key from your Bright Data settings). Pre-configured:
  • Zone → web_unlocker_agent (or whatever you named your zone)
  • URL → dynamic, filled by the agent
  • Format → the agent decides (instructed to always use markdown for clean, readable content)
The tool description tells the agent: "Fetch full content of any URL as clean Markdown. Handles JS-rendered pages, CAPTCHAs, anti-bot. Use for Google search URLs and source pages."
4
Configure the output (last module) - any destination works
The output can go anywhere. Pick what your team uses:
  • Slack → Create a Message (template default, with markdown rendering)
  • Notion → Create a Database Item (store reports in a research database)
  • Airtable → Create a Record
  • Google Sheets → Add a Row (log reports back to a sheet)
  • Gmail → Send an Email (email the report)
  • Discord / Microsoft Teams → equivalent Send Message module
  • Linear / Jira → Create an Issue (turn insights into tickets)
The template ships with Slack as a placeholder. Just map the agent's output ({{4.response}}) to the main content field of whatever destination you pick.
5
Test and activate
Click Run once. Add a test row to your Google Sheet (Research goal + audience). Watch the AI Agent fetch Google, pick sources, call Bright Data multiple times, and produce the final report. Verify the Slack message arrives with proper markdown formatting. Toggle the scenario ON.

Troubleshooting

ProblemSolution
Agent doesn't call the Web Unlocker toolCheck that the tool is properly attached to the AI Agent module. The tool description must clearly say when to use it.
Web Unlocker 403 / timeoutMake sure your Bright Data zone (web_unlocker_agent) exists and is active. For JS-heavy sites, use a zone with rendering enabled.
Agent runs too long / hits timeoutLower recursionLimit in the AI Agent settings (try 100-150 for simpler topics). Or tighten the system prompt: "Stop after 5 source fetches."
Report quality is poorIncrease Reasoning effort to medium or high. Or switch to a stronger model (Claude Opus 4.7 with high reasoning).
Slack formatting brokenThe agent outputs Slack markdown (single asterisks for bold). Make sure Use markdown is enabled in the Slack module.
LLM rate limitsAdd a Sleep module between runs or reduce trigger frequency. Or use Claude Haiku / GPT-5-mini for cheaper calls.

5 nodes: Form Trigger → AI Research Agent (with Claude + Bright Data tool) → Slack (Send Report)

Get n8n template
Works with any input and output: the template uses a Form trigger and Slack output by default, but both are fully swappable. Trigger can be a webhook, chat, schedule, Slack slash command, Airtable / Notion / HubSpot trigger. Output can be Slack, Discord, Teams, Gmail, Notion, Google Sheets, Airtable, SMS, a CRM, or a webhook response.
1
Set up the trigger (first node) - any source works
The trigger can be anything. Pick whatever matches your use case:
  • Form Trigger → public form where users type a research goal (template default)
  • Webhook → POST from any external app
  • Chat Trigger → chat with the agent directly
  • Schedule Trigger → recurring research task
  • Slack / Discord Trigger → slash command
  • Email / Gmail Trigger → trigger from an incoming email
  • Airtable / Notion / HubSpot Trigger → process new records
The template ships with a Form Trigger as a placeholder (fields: Research goal, Target audience). Whatever trigger you use, make sure the output includes those two fields (or adjust the agent input in Step 2).
2
Configure the AI Agent node (the core of this workflow)
The second node is n8n's native AI Agent (LangChain Tools Agent). This is where the autonomous research happens.

The agent system prompt is already pre-configured to:
  • Start with a Google search via web_unlocker
  • Identify the 3-5 most relevant, authoritative sources
  • Fetch each source and synthesize findings
  • Output a Slack-ready structured report (TL;DR, Key findings, Sources, Recommended next step)
The Input is mapped from the trigger (Research goal + Target audience). You can tweak the prompt to specialize the agent (e.g. competitor intel, technical deep-dive, market analysis) or customize the report format.
3
Connect the LLM (Claude, GPT, Gemini...)
The AI Agent needs a chat model. The template uses Anthropic Claude Opus 4.7 (best for agentic reasoning), but you can swap for any supported model:
  • OpenAI → GPT-5, GPT-5-mini, GPT-4o
  • Google Gemini → Gemini 2.0/2.5 Pro
  • Mistral → Mistral Large
  • Groq → Llama 3.3 70B (fast + cheap)
  • Ollama → local models
Click the chat model node, connect your API credentials, and pick the model.
4
Connect Bright Data Web Unlocker as a tool
The AI Agent has a Bright Data Web Unlocker node attached as a tool. This gives the agent "eyes on the web."

First, activate Web Unlocker on Bright Data (if not already done):
  • Log in at brightdata.com
  • In the left sidebar, go to Proxies & Scraping Infrastructure
  • Click Add (or Get started)
  • Select Web Unlocker from the list of zone types
  • Click Start with Web Unlocker or Create zone
  • Name the zone (e.g. web_unlocker_agent) and save
Full docs: Web Unlocker introduction.

Then, in n8n, click the Bright Data tool node and connect your Bright Data credentials (API key from your Bright Data settings). Pre-configured:
  • Zone → web_unlocker_agent (or whatever you named your zone)
  • URL → dynamic, filled by the agent via fromAI
  • Country → dynamic, agent-controlled
The tool description tells the agent: "Fetch full content of any URL as clean Markdown. Handles JS-rendered pages, CAPTCHAs, anti-bot. Use for Google search URLs and individual source pages."
5
Configure the output (last node) - any destination works
The output can go anywhere. Pick what your team uses:
  • Slack → Send Message (template default, mrkdwn rendering)
  • Discord / Microsoft Teams / Telegram → Send Message
  • Email → Gmail / Outlook / SMTP
  • Notion → Create a Page
  • Airtable → Create a Record
  • Google Sheets → Append Row
  • CRM → HubSpot / Salesforce Create Note
  • Respond to Webhook → send the report back to the caller
The agent's report is available in {{ $json.output }} - plug it into the main content field of your chosen destination.

Test and activate

Troubleshooting

ProblemSolution
Agent doesn't call the Web Unlocker toolCheck that the Bright Data node is connected as ai_tool (not in the main flow). The tool description must clearly state when to use it.
Web Unlocker 403 / timeoutMake sure your Web Unlocker zone (web_unlocker_agent) is created and active in your Bright Data dashboard.
Agent loops forever / token spikeSet maxIterations in the AI Agent node settings (default 10). Lower it to 5-7 for simpler research.
Report quality is poorSwitch to a stronger model (Claude Opus 4.7 or GPT-5). Or tighten the system prompt with specific data requirements.
Slack formatting brokenThe agent outputs Slack mrkdwn (*bold*, _italic_). Don't use standard markdown **bold**.
Anthropic rate limitsSwitch to Claude Haiku or GPT-5-mini for cheaper + faster calls. Or add a Wait node between retries.
